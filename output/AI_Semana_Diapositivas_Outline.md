# Outline de Diapositivas: Semana de IA (5 sesiones)

## Slide 1: Portada
- Título: «Introducción práctica a la IA»
- Subtítulo: 5 lecciones para comprender, debatir y crear con IA
- Nombre del docente, fecha

## Lección 1
- Objetivos: diferenciar IA vs automatización; ejemplos cotidianos.
- Definición breve de IA y automatización (dos columnas).
- Ejemplos con íconos: spam, teclado predictivo, mapas, filtros.
- Mini recordatorio de IA generativa y alucinaciones.
- Actividad visual: tarjetas gato/perro → precisión.
- Exit ticket: frase IA vs automatización.

## Lección 2
- Objetivos: ciclo ML, falsos positivos/negativos, sobreajuste.
- Diagrama del ciclo: datos → entrenamiento → prueba → modelo → predicción.
- Tipos de ML: supervisado/ no supervisado/ refuerzo (viñetas + ejemplo cada uno).
- Tabla rápida: FP vs FN con ejemplos (salud, seguridad).
- Gráfico conceptual de sobreajuste (curva entrenamiento vs prueba).
- Exit ticket: define sobreajuste.

## Lección 3
- Objetivos: sesgo y equidad en IA.
- Definición de sesgo + ejemplo de contratación.
- Precisión promedio vs impacto en subgrupos.
- Casos para analizar: crédito, vigilancia, salud (bullet con riesgo principal).
- Mitigaciones: datos balanceados, auditoría, apelación, transparencia.
- Exit ticket: riesgo + mitigación.

## Lección 4
- Objetivos: privacidad y uso de datos.
- Definición de datos personales y minimización.
- Riesgos: fuga, reidentificación, uso secundario.
- Formato de debate (tiempos y roles) + moción propuesta.
- Buenas prácticas: consentimiento revocable, cifrado, opción de borrar.
- Exit ticket: regla de privacidad.

## Lección 5
- Objetivos: diseñar solución con IA responsable.
- Pasos: problema → usuarios → datos → modelo → riesgos/mitigación → métricas.
- Plantilla visual del lienzo de proyecto.
- Ejemplos de ideas (reciclaje escolar, tutor de estudio, alerta de clima).
- Recordatorio de criterios de evaluación del proyecto.
- Exit ticket: riesgo + mitigación del proyecto.

## Cierre
- Resumen de aprendizajes clave.
- Invitación a verificar fuentes y citar cuando se use IA generativa.
